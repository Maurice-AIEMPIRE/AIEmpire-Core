# Max Revenue Operating System (DE AI-Automation)

## Ziel
- Reichweite: taeglich konstante Distribution auf YouTube Shorts + X.
- Conversion: Produkt-First CTA ohne Fake-Einkommensversprechen.
- Lernloop: jede 24h Inhalte nach KPI adaptieren.

## Was noch fehlt (Blocker)
- `OPENAI_API_KEY` (Sora live)
- `YOUTUBE_CLIENT_ID`, `YOUTUBE_CLIENT_SECRET`, `YOUTUBE_REFRESH_TOKEN` (Upload live)
- `STRIPE_SECRET_KEY` (echter Income statt nur Projektion)

## 72h Go-Live Plan
1. Heute: 6 Shorts live + 12 X Posts.
2. Morgen: nur Winning-Patterns verdoppeln, Verlierer pausieren.
3. Tag 3: 1 Offer-Experiment (Preis/Bundle/Lead-Magnet) pro 24h.

## Content-Mix (pro 10 Shorts)
- 4x "How-to" (sofort umsetzbar)
- 3x "Journey" (von Start bis jetzt)
- 2x "Proof" (Screens, Systeme, Fehler + Fix)
- 1x "Contrarian" (Myth Busting)

## Hook-Formel
- "Wenn du [Ziel], dann hoer auf mit [Fehler]."
- "Ich habe [X] Wochen falsch gemacht. Das war der Fix."
- "Copy-Paste diese 3 Schritte, wenn du heute starten willst."

## Monetization-Regeln
- 70% Produkt-CTA, 30% Community-CTA.
- Pro Asset nur 1 CTA.
- Keine unpruefbaren Einkommen-Claims.
- Immer: Nutzen -> Beweis -> Next Step.

## KPI-Gates (taeglich)
- `avg_views_per_hour >= 120`
- `like_rate >= 0.03`
- `comment_rate >= 0.005`

Wenn unterschritten:
- Hook aggressiver
- Script kuerzer
- Angle wechseln
- CTA vereinfachen (1-Wort-Antwort)

## Daily Execution (90 Minuten)
1. 20m: Trend + Kommentare lesen (YouTube/X).
2. 30m: 3 Shorts-Skripte + 6 X Posts bauen.
3. 20m: Render + Publish + Queue.
4. 20m: KPI-Review + naechsten Test setzen.
